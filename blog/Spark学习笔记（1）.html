1、Spark算子分类

Spark 算子大致可以分为以下两类：

  • Transformation变换/转换算子：这种变换并不触发提交作业，完成作业中间过程处理，需要等到有 Action 操作的时候才会真正触发运算（惰性运算）

  • Action行动算子：这类算子会触发 SparkContext 提交 Job 作业，并将数据输出 Spark系统

更细致的可以划分为三类：

  • value数据类型的Transformation算子，这种变换并不触发提交作业，针对处理的数据项是value型的数据

  • key-value数据类型的Transfromation算子，这种变换并不触发提交作业，针对处理的数据项是key-value型的数据对

  • Action算子，这类算子会触发SparkContext提交Job作业

2、value数据类型的Transformation算子　

（1）输入分区与输出分区一对一型

  • map算子

作用：返回一个新的RDD，该RDD由每一个输入元素经过一个函数转换后组成（这个函数需要自己编写，可以非常简单，也可以很复杂） 

val data=sc.parallelize(0 to 4,2)
val m=data.map(x=>x to 3)
m.foreach(println)
/* 
Range 0 to 3
Range 1 to 3
Range 2 to 3
Range 3 to 3
empty Range 4 to 3
*/
println(m.count())//RDD元素数量
//5
  • flatMap算子

作用：类似于map，但是每一个输入元素可以被映射为0或多个输出元素，所以编写函数时应该返回一个序列，而不是单一元素 

val data=sc.parallelize(0 to 4,2)
val fm=data.flatMap(x=>x to 3)
fm.foreach(println)
/*
0
2
3
3
1
2
3
1
2
3
*/
println(fm.count())
// 10

可以看到map()是将一个元素生成一个元素，而flatMap()是将一个元素生成$N$（也可能0）个元素

  • mapPartitions算子

作用：类似于map，但独立地在 RDD 的每一个分片上运行，因此在类型为$T$的 RDD 上 运行时，func 的函数类型必须是Iterator[T] => Iterator[U]。假设有$N$个元素，有$M$个分区， 那么map的函数的将被调用$N$次,而mapPartitions被调用$M$次,一个函数一次处理所有分区 

val data = sc.parallelize(1 to 10,3)

def mapF(x:Int):Int={
  println("调用了mapF方法")
  x
}

def mapParrtitionsF(x:Iterator[Int]):Iterator[Int]={
  println("调用了mapParrtitionsF方法")
  x
}

data.mapPartitions(x=>mapParrtitionsF(x)).count()
/*
调用了mapParrtitionsF方法
调用了mapParrtitionsF方法
调用了mapParrtitionsF方法
*/

data.map(x=>mapF(x)).count()    
/*
调用了mapF方法
调用了mapF方法
调用了mapF方法
调用了mapF方法
调用了mapF方法
调用了mapF方法
调用了mapF方法
调用了mapF方法
调用了mapF方法
调用了mapF方法
*/
  • glom算子

作用：将每一个分区中所有元素形成一个数组，形成新的 RDD 类型是RDD[Array[T]] 

val rdd = sc.parallelize(1 to 16,4)
rdd.glom().collect()
/*
Array[Array[Int]] = Array(Array(1, 2, 3, 4), Array(5, 6, 7, 8), Array(9, 10, 11, 12), Array(13, 14, 15, 16))
*/

（2）输入分区与输出分区多对一型　

  • union算子

作用：对源 RDD 和参数 RDD 求并集后返回一个新的 RDD（使用 union函数时需要保证两个 RDD 元素的数据类型相同，返回的 RDD 数据类型和被合并的 RDD 元素数据类型相同，并不进行去重操作） 

val rdd1 = sc.parallelize(1 to 5)
val rdd2 = sc.parallelize(5 to 10)
val rdd3 = rdd1.union(rdd2)
rdd3.collect()
//Array[Int] = Array(1, 2, 3, 4, 5, 5, 6, 7, 8, 9, 10)
  • cartesian算子

作用：计算两个RDD的笛卡尔积(尽量避免使用) 

val rdd1 = sc.parallelize(1 to 3)
val rdd2 = sc.parallelize(2 to 5)
rdd1.cartesian(rdd2).collect()
// Array[(Int, Int)] = Array((1,2), (1,3), (1,4), (1,5), (2,2), (2,3), (2,4), (2,5), (3,2), (3,3), (3,4), (3,5))

（3）输入分区与输出分区多对多型

  • grouBy算子

作用：分组，按照传入函数的返回值进行分组。将相同的 key 对应的值放入一个迭代器（即将元素通过函数生成相应的 Key，数据就转化为 Key-Value 格式，之后将 Key 相同的元素分为一组） 

val rdd = sc.parallelize(1 to 4)
val group = rdd.groupBy(_%2)
group.collect
// Array[(Int, Iterable[Int])] = Array((0,CompactBuffer(2, 4)), (1,CompactBuffer(1, 3)))

（4）输出分区为输入分区子集型

  • filter算子
作用：过滤，返回一个新的 RDD，该 RDD 由经过 func 函数计算后返回值为 true 的输入元素组成，内 部 实 现 相 当 于 生 成FilteredRDD(this，sc.clean(f))
var sourceFilter = sc.parallelize(Array("xiaoming","xiaojiang","xiaohe","dazhi"))
val filter = sourceFilter.filter(_.contains("xiao"))
filter.collect()
// Array[String] = Array(xiaoming, xiaojiang, xiaohe)
  • distinct算子

作用：对源 RDD 进行去重后返回一个新的 RDD。默认情况下，只有 8 个并行任务来操 作，但是可以传入一个可选的 numTasks 参数改变它 

val distinctRdd = sc.parallelize(List(1,2,1,5,2,9,6,1))
val unionRDD = distinctRdd.distinct() // 不指定并行度
unionRDD.collect()
// Array[Int] = Array(1, 9, 5, 6, 2)
val unionRDD = distinctRdd.distinct(2) // 指定并行度为 2
unionRDD.collect()
// Array[Int] = Array(6, 2, 1, 9, 5)
  • subtract算子

作用：计算差的一种函数，去除两个 RDD 中相同的元素，不同的 RDD 将保留下来 

val rdd = sc.parallelize(3 to 8)
val rdd1 = sc.parallelize(1 to 5)
rdd.subtract(rdd1).collect()
// Array[Int] = Array(8, 6, 7)
  • sample算子

sample(withReplacement, fraction, seed) 作用：以指定的随机种子随机抽样出数量为 fraction 的数据，withReplacement 表示是抽 出的数据是否放回，true 为有放回的抽样，false 为无放回的抽样，seed 用于指定随机数生成器种子 

val rdd = sc.parallelize(1 to 10)
var sample1 = rdd.sample(true,0.4,2)
sample1.collect()
// Array[Int] = Array(1, 2, 2, 7, 7, 8, 9)
var sample2 = rdd.sample(false,0.2,3)
sample2.collect()
// Array[Int] = Array(1, 9)
  • takeSample算子

takeSample(withReplacement, num, seed) 作用：takeSample函数和上面的sample函数是一个原理，但是不使用相对比例采样，而是按设定的采样个数进行采样，同时返回结果不再是RDD，而是相当于对采样后的数据进行Collect，返回结果的集合为单机的数组（直接存在了内存上） 

val rdd = sc.parallelize(1 to 10)
var sample1 = rdd.sample(true,3,2)
println(sample1)
// Array[Int] = Array(8, 10, 9)

（5）Cache型

  • cache算子　　

作用：将 RDD 元素从磁盘缓存到内存。 相当于persist(MEMORY_ONLY)函数的功能

  • persist算子

作用：对 RDD 进行缓存操作。数据缓存在哪里依据 StorageLevel 这个枚举类型进行确定

3、 key-value数据类型的Transfromation算子

（1）输入分区与输出分区一对一

  • mapValues算子

作用：针对（key， value）型数据中的 value 进行 Map 操作，而不对 key 进行处理 

val rdd3 = sc.parallelize(Array((1,"a"),(1,"d"),(2,"b"),(3,"c")))
rdd3.mapValues(_+"|||").collect()
//  Array[(Int, String)] = Array((1,a|||), (1,d|||), (2,b|||), (3,c|||))

（2）对单个RDD或两个RDD聚集

一、单个RDD聚集

  • combineByKey算子

作用：对相同key，把value合并成一个集合
 参数：(createCombiner: V => C, mergeValue: (C, V) => C, mergeCombiners: (C, C) => C) (1)createCombiner：combineByKey会遍历分区中的所有元素，因此每个元素的键要么还没有遇到过， 要么就和之前的某个元素的键相同。如果这是一个新的元素，combineByKey会使用一个叫作createCombiner的函数来创建那个键对应的累加器的初始值
 (2)mergeValue：如果这是一个在处理当前分区之前已经遇到的键，它会使用mergeValue方法将该键的累加器对应的当前值与这个新的值进行合并
 (3)mergeCombiners：由于每个分区都是独立处理的， 因此对于同一个键可以有多个累加器。如果有两个 或者更多的分区都有对应同一个键的累加器， 就需要使用用户提供的mergeCombiners方法将各个分区 的结果进行合并
 

val input = sc.parallelize(Array(("a",88), ("b",95),("a", 91), ("b", 93), ("a", 95), ("b", 98)),2)))
val combine = input.combineByKey((_,1),(acc:(Int,Int),v) => (acc._1+v,acc._2+1), (acc1:(Int,Int),acc2:(Int,Int))=> (acc1._1+acc2._1,acc1._2+acc2._2)) // 将相同 key 对应的值相加，同时记录该 key 出现的次数，放入一个二元组
combine.collect
// Array[(String, (Int, Int))] = Array((b,(286,3)), (a,(274,3)))
val result = combine.map{case (key,value) => (key,value._1/value._2.toDouble)}
//  Array[(String, Double)] = Array((b,95.33333333333333), (a,91.33333333333333))
  • reduceByKey算子

作用：在一个(k,v)的 RDD 上调用，返回一个(k,v)的 RDD，使用指定的 reduce 函数，将相同 key 的值聚合到一起，reduce 任务的个数可以通过第二个可选的参数来设置
 reduceByKey(func, [numTasks])

val rdd = sc.parallelize(List(("female",1),("male",5),("female",5),("male",2)))
val reduce = rdd.reduceByKey((x,y) => x+y)
reduce.collect()
// Array[(String, Int)] = Array((female,6), (male,7))
  • partitionBy算子

作用：对 pairRDD 进行分区操作，如果原有的 partionRDD 和现有的 partionRDD 是一致 的话就不进行分区， 否则会生成 ShuffleRDD，即会产生 shuffle 过程 

val rdd = sc.parallelize(Array((1,"aaa"),(2,"bbb"),(3,"ccc"),(4,"ddd")),4)
rdd.partitions.size
// Int=4
var rdd2 = rdd.partitionBy(new org.apache.spark.HashPartitioner(2))
rdd2.partitions.size
// Int=2

二、两个RDD聚集

  • Cogroup算子

作用：在类型为(K,V)和(K,W)的RDD上调用，返回一个(K,(Iterable,Iterable))类型的RDD
 

val rdd = sc.parallelize(Array((1,"a"),(2,"b"),(3,"c")))
val rdd1 = sc.parallelize(Array((1,4),(2,5),(3,6)))
rdd.cogroup(rdd1).collect()
// Array[(Int, (Iterable[String], Iterable[Int]))] = Array((1,(CompactBuffer(a),CompactBuffer(4))), (2,(CompactBuffer(b),CompactBuffer(5))), (3,(CompactBuffer(c),CompactBuffer(6))))

（3）连接

  • join算子

join(otherDataset, [numTasks]) 作用：在类型为(K,V)和(K,W)的 RDD 上调用，返回一个相同 key 对应的所有元素对在一 起的(K,(V,W))的 RDD 

val rdd = sc.parallelize(Array((1,"a"),(2,"b"),(3,"c")))
val rdd1 = sc.parallelize(Array((1,4),(2,5),(3,6)))
rdd.join(rdd1).collect()
// Array[(Int, (String, Int))] = Array((1,(a,4)), (2,(b,5)), (3,(c,6)))
  • leftOutJoin和 rightOutJoin算子

作用：LeftOutJoin（左外连接）和RightOutJoin（右外连接）相当于在join的基础上先判断一侧的RDD元素是否为空，如果为空，则填充为空。 如果不为空，则将数据进行连接运算，并返回结果

3、Action算子

本质上在 Action 算子中通过 SparkContext 进行了提交作业的 runJob 操作，触发了RDD DAG 的执行

（1）无输出

  • foreach算子

作用：对 RDD 中的每个元素都应用定义的函数操作，不返回 RDD 和 Array， 而是返回Uint 

var rdd = sc.makeRDD(1 to 5,2)
rdd.foreach(println(_))
/*
3
4
5
1 
2
*/

（2）HDFS

  • saveAsTextFile算子

作用：函数将数据输出，存储到 HDFS 的指定目录

  • saveAsObjectFile算子

作用：将分区中的每10个元素组成一个Array，然后将这个Array序列化，映射为(Null，BytesWritable(Y))的元素，写入HDFS为SequenceFile的格式

（3）Scala集合和数据类型

  • collect算子

作用：相当于toArray，toArray已经过时不推荐使用， collect将分布式的 RDD 返回为一个单机的scala Array数组。在这个数组上运用 scala 的函数式操作

  • collectAsMap算子

作用：对(k,v)型的RDD数据返回一个单机HashMap。 对于重复K的RDD元素，后面的元素覆盖前面的元素

  • reduceByKeyLocally算子

作用：先对RDD的整体进行reduce操作，然后再收集所有结果返回为一个HashMap

  • lookup算子

作用：对(k,v)型的RDD操作，返回指定Key对应的元素形成的Seq

  • count算子

作用：返回整个 RDD 的元素个数

  • top算子

作用：返回最大的k个元素
 top（num：Int）（implicit ord：Ordering[T]）：Array[T]

  • reduce算子

作用：将RDD中元素前两个传给输入函数，产生一个新的return值，新产生的return值与RDD中下一个元素（第三个元素）组成两个元素，再被传给输入函数，直到最后只有一个值为止 

val c = sc.parallelize(1 to 10)
c.reduce((x, y) => x + y)
// Int = 55
  • fold算子

作用：折叠操作，aggregate 的简化操作（fold和reduce的原理相同，但是与reduce不同，相当于每个reduce时，迭代器取的第一个元素是zeroValue）
 fold(num)(func)

val c = sc.parallelize(1 to 10)
c.fold(0)(_*_)
// Int = 0
c.reduce(_*_) 
// Int = 3628800
  • aggregate算子

参数：(zeroValue: U)(seqOp: (U, T) ⇒ U, combOp: (U, U) ⇒ U) 作用：将每个分区里面的元素通过 seqOp 和初始值进行聚合，然后用 combine 函数将每个分区的结果和初始值(zeroValue)进行 combine 操作。这个函数最终返回 的类型不需要和 RDD 中元素类型一致 

var rdd = sc.makeRDD(1 to 10,2)
rdd.aggregate(0)(_+_,_+_)
// Int = 55
  • countByKey()算子

作用：针对(K,V)类型的 RDD，返回一个(K,Int)的 map，表示每一个 key 对应的元素个数 

val rdd = sc.parallelize(List((1,3),(1,2),(1,4),(2,3),(3,6),(3,8)),3)
rdd.countByKey
// scala.collection.Map[Int,Long] = Map(3 -> 2, 1 -> 3, 2 -> 1)
